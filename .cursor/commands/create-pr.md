# 创建 PR

## 概览
创建结构清晰的 Pull Request，包含合适的描述、标签和评审者。

## 步骤
1. **准备分支**
   - 确保所有更改已提交
   - 将分支推送到远程
   - 确认分支已与 main 同步

2. **撰写 PR 描述**
   - 清晰概述变更
   - 补充背景和动机
   - 列出任何破坏性变更
   - 若涉及 UI 变更，附上截图

3. **设置 PR**
   - 使用清晰的标题创建 PR
   - 添加合适的标签
   - 指定评审者
   - 关联相关 issues

## PR 模板
- [ ] 功能 A
- [ ] Bug 修复 B
- [ ] 单元测试通过
- [ ] 手动测试完成
